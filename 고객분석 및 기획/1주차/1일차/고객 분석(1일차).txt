DCX
    데이터 기반 고객 경험(Data driven Customer experience)
    디지털 관점에서의 경험
    빅데이터 -> 인사이트
    고객의 사용 맥락 관찰
    기능 품질 프리미엄 경험 의미

    냉장고 -> 누가 열였는지 알 수 없음
        엄마가 열었으면 -> 요리 -> 요리 시간대 추측 가능 -> 맥락파악 -> 새로운 서비스 제공 확장
        워킹맘 -> 아이들이 냉장고 문 열었을 때 -> 냉장고 센서 -> 아이들이 잘 있는지 확인할 수 있음

양적 성장
    데이터 수집 범위 확대
    소셜 분석 기대치가 높기 때문에 분석 기법 발전시켜 분석 정확도 올리는 것 필요

질적 성장
    소셜 데이터 분석 및 해석, 도메인 지식 공부

소셜 데이터 분석 방법
1. 소셜 데이터 분석 사이트 활용
    분석 타겟 정할 시 트렌드 분석은 필수
    데이터 수집, 전처리, 분석 기법 활용 등으로 트렌드 파악 가능
    사이트 별 특징을 미리 이해하고 있으면 분석에 앞서 미리 계획이 가능(사이트마다 서비스가 다름)
    설득력+분석력 높이는데 도움

2. 대표적인 소셜 데이터 분석 사이트
    썸트렌드
    네이버 데이터랩
    구글 트렌드
    카카오 데이터 트렌드(생긴지 별로 안되서 기간 짧음)

텍스트 마이닝
    워드클라우드 : 주요 키워드 추출, 직관적
    군집 분석
    이상치 분석
    분류
    네트워크 분석, 토픽모델/트렌드 분석, 표절분석(유사도 계산)

DCX 가치 창출 방법론
    경영 문제 -> 고객(페르소나) 이해 -> 난제(pain point) 발굴 -> 해결책(solution)
    data를 먼저 확보하고 시작 -> 고객을 data로부터 클러스팅 : 여러 개의 고객을 만들 수 있음
    pain point 분류를 통해서도 인사이트를 얻을 수 있다 -> 공통적인 pain point가 있는지 확인 

문제정의 - 데이터 준비 - 데이터 분석 - 결과
    문제정의 : 타겟 키워드 분류
        1) 타겟 사이트 중심
            자주 사용하는 사이트 -> 남기는 글, 특정 사람에게 한정되어 있는 데이터
        2) 키워드 중심
            언어 중심 검색포털 크롤링 -> 라이프 스타일 파악

        선행연구 -> 브레인스토밍 -> 카테고리 도출 -> 부적합 카테고리 소거 -> 카테고리 수렴 -> 최종 검증
    
    데이터 준비 : 크롤링 - 광고제거 - 텍스트 전처리 - 유효 데이터 도출
        데이터 전처리
            1) 광고성 글 제거
            2) 맥락 무관 문구 처리 & 형태소 변환

    데이터 분석
        고객 클러스터링 : 데이터 군집화 - actor 도출 - 군집별 LDA(토픽별)
            Ward clustering 알고리즘 사용 -> 유사한 의미와 맥락을 가진 텍스트 데이터끼리 군집화한 세그먼트 도출
            
        action 도출 및 CAM 작성 : action 도출 및 고객액션맵 - 기회영역(액션 중요도로 구분) 및 4D-CX - 인사이트와 페르소나 도출
            TF-IDF 결과 확인(사용 빈도수)
            모든 클러스터를 정의하고 분석 -> 군집으로 나눔
            중요도, 만족도 계산

    결과 : 데이터 기반 아이디어 워크숍 및 결과 도출, 시장조사를 기반으로 한 제품설계, 결과정리 및 피드백
        인사이트 도출
        단기적/장기적 개선 -> 어떤 경험을 하기를 원하는지, 충족시키기 위해서는 어떤 컨셉이 필요한지
        페르소나 + 니즈 + 컨셉 => 제품

주제 선정 -> 세부적으로 들어갈 때 트렌드 분석을 통해 진짜 필요한 서비스인지 확인
