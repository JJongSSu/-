DCX 프로세스
    1. data defining
        키워드, 세그먼트 선정 -> 소셜 리스닝
        키워드 기반 세그먼트 탐색, 크롤링 채널 탐색 -> 크롤링 및 전처리
    2. data sensing
        고객의 데이터를 sensing
        마이크로 세그먼트 도출(actor 도출) -> clustering
        action 맵 -> LDA
    3. data shaping
        
    타겟 디자인
        데이터 수집 목적, 확보 방법(사이트, 키워드), 양적/질적 타당성 검토(키워드 검색량 트렌드, 연관어 분석)
    
    데이터 수집
        크롤링 : 수집기간, 이유, 수집 조건(네이버 view, 블로그, 카페)
        -> 결과확인

    데이터 전처리
        1차 : 광고성 데이터 제거, 자연어 데이터 정제
        2차 : 목적과 무관한 데이터 제거

    페르소나 분석(actor 분석)
        각각의 actor들이 어떤 맥락으로 되어 있는지 확인

    각 actor들의 pain point를 모아보면 대분류로 구분할 수 있음
